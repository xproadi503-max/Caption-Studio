# Caption Studio — Zeemo-style Auto Caption App (Flutter)

Auto-generates animated captions for videos: pick a video → AI transcribes
speech with word-level timing → pick an animated caption template →
preview → export final video with captions burned in.

## What's included
```
lib/
  models/         caption word/line/style/template data classes
  services/        transcription (AssemblyAI), audio extraction, ASS subtitle
                    generation, video export (FFmpeg burn-in)
  providers/       app state (ChangeNotifier)
  screens/         Home → Template Picker → Preview → Caption Editor → Export
  widgets/         live animated caption overlay for the preview/editor screens
  main.dart
pubspec.yaml
```
**30+ caption templates** are predefined in `lib/models/caption_templates.dart`,
covering every animation style (word-highlight/karaoke, pop, bounce,
typewriter, slide-up, fade, static) crossed with a range of color themes
(neon, pastel, gold, monochrome, etc — similar in spirit to Zeemo/CapCut's
template libraries). Add more by appending entries to that list — no
other code changes needed.

**Caption editor (new):** from the Preview screen, tap "Edit caption text
& position" to:
- Drag any caption line directly on the video to place it exactly where
  you want (per line, not just top/center/bottom)
- Tap a line in the list to jump the video to that moment and select it
- Edit the text of any line inline to fix transcription mistakes
- Tap the reset icon next to a line to clear its custom position

These edits are respected in the final exported video (positions and
corrected text both get baked into the burned-in captions).

## 0. Set your API key once (optional but recommended)
Open `lib/screens/home_screen.dart` and find this near the top:
```dart
const String kDefaultApiKey = '';
```
Paste your AssemblyAI key between the quotes (see step 3 below to get
one). The app will then pre-fill it automatically every time — you
won't need to type it in the app each time. This stays local to your
own copy of the code; never commit a real key to a public repo.

## 1. Setup (do this first, only once)

You only received the `lib/` source + `pubspec.yaml` — not the native
Android/iOS project scaffolding (that's generated by Flutter itself and
not meant to be hand-written). Steps:

1. Install Flutter SDK if you haven't: https://docs.flutter.dev/get-started/install
2. Create a new empty Flutter project scaffold:
   ```
   flutter create caption_studio
   cd caption_studio
   ```
3. Delete the generated `lib/` folder and `pubspec.yaml` in your new
   project, then copy in the `lib/` folder and `pubspec.yaml` from this zip.
4. Install dependencies:
   ```
   flutter pub get
   ```

## 2. Permissions setup

**Android** — add to `android/app/src/main/AndroidManifest.xml` (inside `<manifest>`, before `<application>`):
```xml
<uses-permission android:name="android.permission.READ_MEDIA_VIDEO"/>
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>
<uses-permission android:name="android.permission.INTERNET"/>
```
Also set `minSdkVersion 24` or higher in `android/app/build.gradle`.

**iOS** — add to `ios/Runner/Info.plist`:
```xml
<key>NSPhotoLibraryUsageDescription</key>
<string>Needed to pick and save videos</string>
<key>NSPhotoLibraryAddUsageDescription</key>
<string>Needed to save your exported captioned video</string>
```

## 3. Get an AssemblyAI API key (free tier available)
Sign up at https://www.assemblyai.com/ → dashboard → copy your API key.
Paste it into the app's home screen (it's saved locally on-device via
`shared_preferences`, never hardcoded in source).

## 4. Run it
```
flutter run
```

## 5. Notes on FFmpeg
This project uses `ffmpeg_kit_flutter_new` (a maintained fork) instead of
the original `ffmpeg_kit_flutter`, which was discontinued in 2025. It
supports Android, iOS, macOS, and Windows, and includes subtitle burn-in
(`ass` filter) which is what powers the export step.

## 6. Auto-build APK with GitHub Actions (no Codespaces/terminal needed)

This project includes `.github/workflows/build-apk.yml`. Once you push
this repo to GitHub, it automatically:
1. Sets up Flutter + Java
2. Generates the Android scaffold and merges in this project's `lib/`
   and `pubspec.yaml`
3. Adds the required Android permissions
4. Runs `flutter build apk --release`
5. Uploads the finished APK as a downloadable artifact

**To get your APK:**
- Push/upload this repo to GitHub (see step 7 below)
- Go to the **Actions** tab → click the latest workflow run (or click
  "Run workflow" manually if it hasn't triggered yet)
- Wait for it to finish (green checkmark, usually 5-10 min)
- Scroll down to **Artifacts** → download `caption-studio-apk` → it's a
  zip containing `app-release.apk` → install that on your Android phone

No local Flutter install, no Codespaces terminal commands required —
just push and download.

## 7. Uploading this project to GitHub
From inside your project folder (after step 1-4 above):
```
git init
git add .
git commit -m "Initial commit: Caption Studio"
git branch -M main
git remote add origin https://github.com/<your-username>/<your-repo>.git
git push -u origin main
```
Or, without any command line: create a new repo on github.com, click
**"uploading an existing file"** on the empty repo page, and drag the
whole project folder in.

Make sure you have a `.gitignore` (Flutter's default `flutter create`
already gives you one) so build artifacts and your API key preferences
aren't committed.

## Known limitations / next steps
- Live preview animation (Flutter widgets) and the exported video's
  animation (ASS/FFmpeg) are two separate implementations tuned to look
  similar — not pixel-identical. This is normal for this kind of pipeline.
- Long videos = longer AssemblyAI processing time and larger FFmpeg export
  time. Consider showing a "this may take a few minutes" note for videos
  over ~2 minutes.
- Multi-speaker auto-styling (like Zeemo's speaker-based template
  assignment) is not implemented yet — `speaker` is captured on each
  `CaptionWord` if you want to extend this.
